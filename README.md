
# Short Audiobook Director Agent

## 项目目标

本项目是一个面向**短篇中文故事**的有声书导演 Agent 原型。

它的目标不是简单把文字读出来，而是让 Agent 先理解文本中的旁白、对白、角色和基本语境，再生成适合 TTS 使用的有声书制作脚本，最后调用 TTS 模型生成一版可播放的简易有声读物音频。

当前项目聚焦 300-2000 字左右的中文短篇文本，例如：

- 儿童故事
- 寓言故事
- 小学课文
- 绘本故事
- 简短分角色文本

本项目当前不追求长篇小说、商业级音质、复杂 UI、配乐音效或完整有声剧制作。

---

## 核心链路

项目整体链路如下：

```text
输入短篇中文文本
→ 文本清理与分段
→ 识别旁白、对白、引号类型
→ 识别说话角色
→ 生成适合 TTS 的文本片段
→ 建立角色声音档案
→ 生成 TTS 指令
→ 调用 MOSS-TTS-Nano 合成语音
→ 拼接生成简易有声读物 Demo
````

其中，最重要的不是单纯生成音频，而是中间的“导演层”：

```text
谁在说话？
哪部分是旁白？
哪部分是真实对白？
哪些引号只是强调词或概念？
不同角色应该使用什么音色？
这句话应该以什么语气被朗读？
当前 TTS 模型能执行哪些控制，哪些只是导演建议？
```

## 当前 TTS 后端

当前 v0.1 使用 **MOSS-TTS-Nano** 作为 TTS 后端。

该模型已经能够完成基本的中文文本转语音，但控制能力有限。

当前主要可控能力是：

```text
1. 通过 prompt_audio_path 选择参考音频，从而影响音色
2. 通过采样参数影响生成稳定性和自然度
3. 通过后处理控制音频片段之间的停顿
```

当前不能直接精确控制：

```text
1. 情绪
2. 语速
3. 音高
4. 音量
5. 复杂表演风格
```

因此，本项目需要区分：

```text
导演意图：Agent 对情绪、语气、角色表现的理解
实际 TTS 参数：MOSS-TTS-Nano 当前真正能执行的参数
```

当前阶段的策略是：

```text
先让 LLM 生成导演意图
再根据导演意图选择合适的参考音频
最后用 MOSS-TTS-Nano 生成语音
```

## 项目原则

### 1. 不把 story_parser 写成万能正则

中文文本中的引号不一定都是对白。

例如：

```text
桂花成熟时，就应当“摇”。
童年时代的“摇花乐”。
```

这里的引号是强调或概念，不是人物对白。

因此，规则 parser 不应该简单地把所有引号都当成 dialogue。

更合理的思路是：

```text
规则层：负责文本切分、引号位置提取、基础结构整理
LLM 层：负责判断引号类型、说话人和语义关系
程序层：根据判断结果生成最终可用于 TTS 的 segments
```

---

### 2. segment 是最终送入 TTS 的最小单位

每个最终 segment 应该只对应一个说话人和一个音色。

例如：

```text
小松鼠笑着递给他一个篮子，说：“当然来得及，只要你愿意开始。”
```

不能把整句作为一个 TTS segment。

应该拆成：

```text
旁白：小松鼠笑着递给他一个篮子，
小松鼠：当然来得及，只要你愿意开始。
```

这样后续才能实现：

```text
旁白用旁白音色
角色对白用角色音色
```

---

### 3. 先做最小闭环，再逐步增强

当前优先级：

```text
第一优先：文本能被正确读取、分段、结构化
第二优先：旁白和明显对白能被区分
第三优先：明显说话人能被识别
第四优先：能生成角色声音档案和 TTS 指令
第五优先：能调用 MOSS-TTS-Nano 生成音频
```

暂时不要优先做：

```text
1. Web UI
2. 背景音乐
3. 环境音效
4. 长篇小说支持
5. 商业级音质优化
6. 大规模语音库管理
```

## 建议模块分工

```text
text_loader.py
读取文本，做基础清理。

story_parser.py
进行段落切分、引号提取、基础结构解析。

llm_story_resolver.py
使用 LLM 判断引号类型、说话人、是否为真实对白。

segment_builder.py
生成最终适合 TTS 的 segments。

character_analyzer.py
根据文本和 segments 提取角色，并生成角色声音档案。

tts_instruction_generator.py
根据角色、文本和导演意图生成 TTS 指令。

tts_engine.py
调用 MOSS-TTS-Nano 生成语音片段。

audio_stitcher.py
将语音片段按顺序拼接成最终音频。

report_generator.py
输出 Markdown / JSON 报告，方便检查和汇报。
```

## Claude Code 工作要求

为了避免项目偏离主线，使用 Claude Code 时请遵守：

```text
1. 每次只实现一个小模块
2. 每次修改后都要能运行
3. 不要一次性生成全部功能
4. 不要过度针对某一篇文本写死规则
5. 不要随意修改无关文件
6. 不要把所有逻辑塞进 run.py
7. 不要把本项目代码写进 MOSS-TTS-Nano 源码目录
```

推荐开发顺序：

```text
1. 创建项目骨架
2. 实现文本读取
3. 实现段落切分和引号提取
4. 实现基础 segment 构建
5. 接入 LLM 做语义判断
6. 生成角色声音档案
7. 生成 TTS 指令
8. 调用 MOSS-TTS-Nano
9. 拼接音频
10. 整理 Demo 和报告
```

## 项目边界

本项目当前只是一个原型，目标是验证：

```text
文本理解 → 声音导演 → TTS 生成 → 音频输出
```

这个最小闭环是否可行。

当前不要求做到：

```text
1. 完美识别所有说话人
2. 完美处理所有中文课文
3. 完美情绪控制
4. 商业级配音质量
5. 完整有声剧制作
```
